# Translations

This directory holds Qt Linguist translation files for the editor.

- **`opencut_vi.ts`** — Vietnamese. This is the **source** language: every
  `self.tr("…")` call in the codebase passes the Vietnamese string as its
  literal argument, and the `.ts` file is regenerated by scanning the
  source tree with `pyside6-lupdate`.
- **`opencut_en.ts`** — English translation of the Vietnamese source.
- **`opencut_*.qm`** — binary compiled versions loaded at runtime by
  `app.infrastructure.translation_manager`.

## Regenerating translations

After adding or changing any `self.tr("…")` call, run:

```bash
./scripts/update_translations.sh
```

This runs `pyside6-lupdate` to extract translatable strings into the `.ts`
files and `pyside6-lrelease` to compile them into `.qm` binaries. A CI
check in the follow-up PR will verify the `.ts` files are up to date.

## Language selection at runtime

The `translation_manager` resolves the active language from (highest
priority first):

1. The `language` argument passed to `install_translators`.
2. The `OPENCUT_LANG` environment variable (`vi` or `en`).
3. The system locale, if it matches a supported language.
4. `vi` as the hard-coded default.

A user-facing preferences panel backed by a JSON settings store is
planned for Sprint 1–2 (ROADMAP M0).

## Supported languages (v1.0)

| Code | Language     | Completeness target |
|------|--------------|---------------------|
| vi   | Tiếng Việt   | 100% (source)       |
| en   | English      | ≥ 90%               |

Additional languages can be added after v1.0 by creating a new
`opencut_<code>.ts` file and appending the code to
`SUPPORTED_LANGUAGES` in `app/infrastructure/translation_manager.py`.
